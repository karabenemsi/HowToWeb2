Heute nicht behandelte Elemente:

  • abbr

    <abbr>

    Definiert eine Abkürzung oder ein Akronym Bsp: "ASAP", "MfG", "WHO"

    Gibt Browsern, Suchmaschinen und Übersetzern hilfreiche Informationen

    The <abbr title="World Health Organization">WHO</abbr> was founded in 1948.
  • address

    <address>

    Definiert die Kontaktinformationen für den Artikel/die Seite. Im <article>angegeben definiert er die Informationen für den Artikel, sonst für die Seite.

    <address>
  Written by <a href="mailto:webmaster@example.com">Jon Doe</a>.<br> 
  Visit us at:<br>
  Example.com<br>
  Box 564, Disneyland<br>
  USA
</address>
  • area

    <area>

    Beschreibt einen Bereich in einer image-map das Element ist immer in einer <map> verschachtelt.

    <img src="planets.gif" width="145" height="126" alt="Planets"
  usemap="#planetmap">

<map name="planetmap">
  <area shape="rect" coords="0,0,82,126" href="sun.htm" alt="Sun">
  <area shape="circle" coords="90,58,3" href="mercur.htm" alt="Mercury">
  <area shape="circle" coords="124,58,8" href="venus.htm" alt="Venus">
</map>
  • article

    <article>

    Der <article>tag spezifiziert unabhängingen, selbstständigen Inhalt. Ein Artikel sollte ohne Zusätze Sinn ergeben und es sollte moglich sein ihn vom Rest der Seite trennnen zu können.

    <article>
  <h1>Google Chrome</h1>
  <p>Google Chrome is a free, open-source web browser developed by Google.</p>
</article>
  • aside

    <aside>

    Der <aside>tag definiert Inhalt abseits von dem ihn enthaltenen Umfeld. Er sollte inhaltlich zur Umgebung passen.

    <p>My family and I visited The Epcot center this summer.</p>

<aside>
  <h4>Epcot Center</h4>
  <p>The Epcot Center is a theme park in Disney World, Florida.</p>
</aside>

  <source src="horse.mp3" type="audio/mpeg">
  Your browser does not support the audio tag.
</audio>

    <base>

    Definiert eine Standarturl und ein Standarttarget für alle Links auf der Seite

  <img src="stickman.gif" width="24" height="39" alt="Stickman">

  • bdi

    <bdi>

    BDI steht fürBi-Directional Isolation. Der<bdi>tag isoliert einen Teil eines Textes der bei der Änderung der Textrichtung anders formattiert wird. Seine Textrichtung wird beibehalten

    <ul>
  <li>User <bdi>hrefs</bdi>: 60 points</li>
  <li>User <bdi>jdoe</bdi>: 80 points</li>
  <li>User <bdi>????</bdi>: 90 points</li>
</ul>
  • bdo

    <bdo>

    BDO steht für Bi-Directional Overwrite. Damit wird die die aktuelle Textrichtung überschrieben

    <bdo dir="rtl">
  This text will go right-to-left.
</bdo>
  • blockquote

    <blockquote>

    Ein Abschnitt, von einer anderen Quelle zitiert wird.

    <blockquote>
  Niemand hat vor eine Mauer zu bauen - Bob der Baumeister
</blockquote>
  • button

    <button>

    Definiert einen klickbaren Knopf. Er unterscheidet sich von einem Knopf aus einem <input>-Element dadurch, dass hier auch Texte oder Bilder eingefügt werden können.

    <button type="button">Click Me!</button>
  • canvas

    <canvas>

    Auf einem Canvas kann live gemalt werden. Dies geschieht per JavaScript.

    <canvas id="myCanvas"></canvas>

<script>
  var canvas = document.getElementById("myCanvas");
  var ctx = canvas.getContext("2d");
  ctx.fillStyle = "#FF0000";
  ctx.fillRect(0, 0, 80, 80);
 </script>

    <cite>

    Definiert den Namen/Titel eines Werks (z.B. Ein Bild, ein Lied ...)

  • code

  • data

    <data>

    The <data> tag links the given content with a machine-readable translation. This element provides both a machine-readable value for data processors, and a human-readable value for rendering in a browser.

    <ul>
  <li><data value="21053">Cherry Tomato</data></li>
  <li><data value="21054">Beef Tomato</data></li>
  <li><data value="21055">Snack Tomato</data></li>
</ul>
  • datalist

    <datalist>

    The <datalist> tag specifies a list of pre-defined options for an <input> element. The <datalist> tag is used to provide an "autocomplete" feature on <input> elements. Users will see a drop-down list of pre-defined options as they input data. Use the <input> element's list attribute to bind it together with a <datalist> element.

    <input list="browsers">

<datalist id="browsers">
   <option value="Internet Explorer">
   <option value="Firefox">
   <option value="Chrome">
   <option value="Opera">
   <option value="Safari">
</datalist>
  • dd

    <dd>

    The <dd> tag is used to describe a term/name in a description list. The <dd> tag is used in conjunction with <dl> (defines a description list) and <dt> (defines terms/names). Inside a <dd> tag you can put paragraphs, line breaks, images, links, lists, etc.

    <dl>
  <dt>Coffee</dt>
  <dd>Black hot drink</dd>
  <dt>Milk</dt>
  <dd>White cold drink</dd>
</dl>
  • details

    <details>

    The <details> tag specifies additional details that the user can view or hide on demand. The <details> tag can be used to create an interactive widget that the user can open and close. Any sort of content can be put inside the <details> tag. The content of a <details> element should not be visible unless the open attribute is set.

    <details>
  <summary>Copyright 1999-2014.</summary>
  <p> - by Refsnes Data. All Rights Reserved.</p>
  <p>All content and graphics on this web site are the property of Refsnes</p>
</details>
  • dfn

    <dfn>

    The <dfn> tag represents the defining instance of a term in HTML. The defining instance is often the first use of a term in a document. The nearest parent of the <dfn> tag must also contain the definition/explanation for the term inside <dfn>.

    <p><dfn>HTML</dfn> is the standard markup language for creating web pages.</p>
  • dialog

    <dialog>

    The <dialog> tag defines a dialog box or window. The <dialog> element makes it easy to create popup dialogs and modals on a web page.

    <th>January <dialog open>This is an open dialog window</dialog></th>

  • figcaption

    <figcaption>

    The <figcaption> tag defines a caption for a <figure> element. The <figcaption> element can be placed as the first or last child of the <figure> element.

    <figure>
  <img src="img_pulpit.jpg" alt="The Pulpit Rock" width="304" height="228">
  <figcaption>Fig1. - A view of the pulpit rock in Norway.</figcaption>
</figure>
  • figure

    <figure>

    Das <figure> -Tag legt in sich geschlossene Inhalte wie Illustrationen, Diagramme, Fotos, Codelisten usw. fest. Während der Inhalt des <figure> -Elements mit dem Hauptfluss zusammenhängt, ist seine Position unabhängig vom Hauptfluss, und wenn es entfernt wird, sollte dies den Fluss des Dokuments nicht beeinflussen.

    <figure>
  <img src="img_pulpit.jpg" alt="The Pulpit Rock" width="304" height="228">
</figure>
